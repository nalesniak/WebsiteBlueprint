Create a multi-page website on GitHub Pages.

If you need text, use some lorem ipsum lipsum.com

Each file in your site, including images, should have a file name containing only lowercase letters and numbers (use underscores (_) instead of spaces) to make your life easier. Use the attached starter code to get started with each. To link from one file to another, use the link tag (*a*) and change it so href="index.html" to link to the index file or "href="about.html" to link to the About page. That being done, add some content, formatting, and additional pages. Look at the sites below for reference, and embellish your site by adding images, formatting text, using several different types of tags, and adding at least one more page with content of your choosing. Please keep in mind that this page is publicly accessible, so you should not put pictures of yourself or other personally identifiable information on this site!

REQUIREMENTS
  • Some text written or selected by you
  • Images on every page that are resized using the height or width html attribute. Be sure the URL does not begin with file:// as that will not display correctly on the live website https://htmlreference.io/element/img/
Links https://www.w3schools.com/html/html_links.asp
  • at least 1 external link (i.e., to a webpage that is not yours) on each page
  • index.html and at least 2 other html files with content on them
  • internal links between index.html and your other pages
External, Internal, and Inline CSS https://www.w3schools.com/css/css_howto.asp
  • several tags with inline style added
  • external style sheet with CSS rules
  • style section in the head of at least one of the pages
  • All three kinds of CSS selectors https://www.w3schools.com/css/css_selectors.asp
  • select by id, class , and tag somewhere in the style tags and/or an external style sheet
  • all of these tags: *img* *a* *p* *html* *body* *head* *title* (*h1*, *h2*, etc)
  • all of these css attributes: font-family, background-color, color, font-size
  • additional tags and CSS attributes chosen by you from the resources above or elsewhere. There should be a variety to get full credit.
PUBLISHING
    To make your website accessible, go to your repository page, then click Settings -> Pages and choose a branch under "Source" (probably "main") and click "Save". It will now say "Your site is ready to be published at [URL]". It may take ten minutes or so for this to change to "Your site is published at [URL]" which means your site is live and accessible at that URL.
Submit as a link to your GitHub pages site AND a link to your repository. The URL will be username.github.io/repositoryname and the repository URL will be github.com/username/repositoryname
